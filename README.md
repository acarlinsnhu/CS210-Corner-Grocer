# CS 210 Corner Grocer Project

## Project Summary

This project is a C++ program for the Corner Grocer scenario. The program reads a list of grocery items from an input file and counts how many times each item appears. It gives the user a menu where they can search for one item, print the full list of item frequencies, or view the results as a histogram. The problem it solves is helping a small grocery store review item purchase patterns without counting everything by hand.

## What I Did Well

One part I think I did well was keeping the menu easy to follow. The user can pick from clear options, and the program gives useful results based on that choice. I also think the file reading and frequency counting worked well because the program was able to take the input file and turn it into organized data. Getting the search option to return the correct count for specific items also showed that the main logic was working properly.

## Where I Could Enhance the Code

I could improve the program by adding stronger input validation. Right now, the program works for the expected menu choices, but it could be made better by handling incorrect input more smoothly. I could also separate more of the logic into smaller functions so the main part of the program is easier to read. These changes would make the program more efficient to update, easier to test, and safer for a user who enters something unexpected.

## Most Challenging Part

The most challenging part was working with the file input and making sure each item was counted correctly. It took some checking to make sure the program was reading the file the right way and storing the results properly. I overcame this by testing the program with known items from the file and confirming that the counts matched what I expected. Going forward, I would use class resources, C++ documentation, and examples from previous projects as part of my support network.

## Transferable Skills

The skills from this project can transfer to other programming work because file handling, user menus, loops, and data organization show up in many programs. This project also helped me practice writing code that solves a real problem instead of just completing small exercises. Being able to read data, process it, and present it in a useful way is something I can use in future courses and projects.

## Maintainability, Readability, and Adaptability

I made the program maintainable and readable by using clear names and keeping the program organized around the main tasks it needed to perform. The menu structure also makes the program easier to understand because each option has a clear purpose. The program could be adapted later by adding more reports, changing the input file, or expanding the item tracking features without having to completely rewrite the project.
